# 🔄 Laços de Repetição (Loops) em JavaScript

Esta seção reúne os exercícios desenvolvidos durante meus estudos sobre **laços de repetição (loops)** em JavaScript.

Os loops são estruturas fundamentais da programação, pois permitem executar um mesmo bloco de código várias vezes, evitando repetições desnecessárias e tornando os programas mais eficientes.

---

# 🎯 Objetivo

Aprender a utilizar estruturas de repetição para automatizar tarefas, controlar a execução de código e resolver problemas que exigem repetições.

---

# 📚 Conceitos estudados

Durante os exercícios pratiquei:

- Estrutura `while`
- Controle de repetição
- Variáveis de controle (contadores)
- Condições de parada
- Acumuladores
- Validação de entradas do usuário
- Repetição de ações até que uma condição seja satisfeita

---

# 🧠 O que aprendi

## ✅ Estrutura `while`

Aprendi que o `while` executa um bloco de código enquanto uma condição for verdadeira.

Exemplo:

```javascript
while (condicao) {
  // código executado repetidamente
}
```

Também compreendi a importância de atualizar a condição para evitar loops infinitos.

---

## ✅ Contadores

Utilizei variáveis para controlar quantas vezes um bloco de código deveria ser executado.

Exemplos:

- Contagem de 1 até 10
- Contagem regressiva
- Exibição de mensagens repetidas
- Tabuada de um número

---

## ✅ Acumuladores

Aprendi a utilizar variáveis para armazenar resultados ao longo das repetições.

Esse conceito foi aplicado em exercícios como:

- Soma dos números de 1 até 10
- Contagem de elementos
- Cálculo de médias

---

## ✅ Repetição baseada em condições

Pratiquei situações em que o programa continua executando até que o usuário forneça uma informação válida.

Exemplo:

- Sistema de senha
- Menus interativos
- Validação de entradas

# ⚠️ Cuidados importantes

Durante os estudos aprendi que alguns erros podem causar problemas no programa, como:

- Esquecer de incrementar o contador;
- Criar loops infinitos;
- Definir condições de parada incorretas;
- Inicializar variáveis com valores inadequados.

Compreender esses detalhes foi essencial para construir programas mais seguros e organizados.
